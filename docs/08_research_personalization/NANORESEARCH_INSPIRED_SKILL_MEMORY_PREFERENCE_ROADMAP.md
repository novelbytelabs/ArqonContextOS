## Purpose

This document records the post-share roadmap inspired by NanoResearch-style tri-level co-evolution: reusable procedural skills, durable memory, and preference adaptation.

It does not expand Science Monkeys v0.1 Routes 001. Routes 001 remains a command-route layer only.

## Doctrine

Build it the Arqon way:

```text
No hidden memory as truth.
No ungated self-evolution.
No automatic paper/spec generation from weak evidence.
No preference learning without auditability.
No promotion without Human approval.
```

## Mapping

```text
Skill Bank        → Arqon Skill Registry / Procedure Bank
Memory Module     → ContextBus user/project/flow/finding memory artifacts
Planner Policy    → governed, auditable planning preference records
Orchestrator      → ArqonMonkeyOS brokered flow orchestration
Research Pipeline → Science Monkeys + Code Monkeys lifecycle
```

Routes 001 must preserve enough structure for future extraction:

- role
- phase
- flow_id
- artifact_type
- parent references
- evidence level
- source refs
- allowed claims
- forbidden claims
- failure mode
- decision status

Routes 001 must not implement hidden personalization, policy learning, or autonomous research-paper generation.
