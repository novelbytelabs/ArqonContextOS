## ContextOS

Repo-backed operating layer for GPT team context, messages, notes, flows, artifacts, evidence, and governance.

## Flow

Official workflow container. Replaces run at the user level.

## Flow name

Human-friendly alias such as `flow00`.

## Flow ID

Canonical ID such as `FLOW-2026-0001`.

## Note

Non-official saved context.

## Message

Non-official directed communication.

## Artifact

Official work product inside a flow.

## Spec Kit

GitHub specification-driven development workflow/tooling.

## Dossier

ContextOS pre-spec context artifact.

## Gate

Logical approval level or advancement condition.

## Phase flow

Flow where PM remains closely involved.

## Milestone flow

Flow where Coder/Helper work through PM-defined milestone chunk.

## Audit flow

Flow where Auditor gates milestone/risk/claim points.

## Helper/Codex

Local VS Code/Codex execution worker.

## Advancement

Movement from one flow state to another, usually through `/adv-flow`.
