## Purpose

Preserve a future benchmark track for testing Arqon Zero / MonkeyOS against CyberGym when the system is mature enough.

This is a roadmap document only. It does not activate CyberGym work now.

## Benchmark Goal

Evaluate Arqon Zero / MonkeyOS on vulnerability reproduction tasks while measuring both task success and truth-governance quality.

## Why CyberGym Matters

CyberGym is useful because it stresses repository exploration, vulnerability understanding, proof-of-concept construction, test execution, false-success avoidance, and evidence discipline.

That aligns with the Arqon thesis that agentic work must be governed by proof and audit, not just generated answers.

## Proposed Metrics

### CyberGym Reproduction Score

Did the system produce a working vulnerability reproduction?

### Truth-Governance Score

Did the system avoid fake success, missing logs, skipped tests, hallucinated proof, promotion without evidence, and incomplete reproduction packets?

### Learning Score

Does the system improve after repeated tasks through future Skill/Memory/Preference layers?

## Proposed Phases

### Phase 1: CyberGym Lite

Run 20 to 50 selected tasks.

Measure baseline success, false-positive rate, time, cost, and audit quality.

### Phase 2: Cyber Prover Pipeline

Add a controlled pipeline:

Explore repo → identify target area → propose proof path → generate test → run test → minimize proof → audit → human review

### Phase 3: Compare Against Public Baselines

Compare against public CyberGym baselines before making any MDASH-style comparison.

### Phase 4: MDASH-Style Comparison

Only compare against MDASH-like claims when the same task split, scoring, budget, timeout, repo state, and proof criteria are available.

## Non-Claims
